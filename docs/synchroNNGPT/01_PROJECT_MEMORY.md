## Purpose

AISAは企業診断ではなく、一つの仕事・タスクを診断します。対象タスクの状況、制約、期待成果を整理し、実行可能な選択肢を提示します。

## Proposal Model

AISAは原則として次の三つの観点を提示します。

1. 費用優先型
2. 時間短縮優先型
3. AI活用優先型

各観点では、無料、低価格、本格導入の順に、現実的な選択肢を示します。

## Collaboration Model

- 人とAIの役割を固定しない。
- その時点で最も価値を生み出す主体が役割を担う。
- 人とAIを競争、比較、分断しない。
- 「役割ではなく、価値で協働する」を中心理念とする。

## Repository Rules

- mainへの通常の直接コミットは禁止する。
- 変更は作業ブランチとPull Requestを通す。
- AgreeとAlignをレビューの共通語とする。
- 未確定事項は理念を独自に変更せず、TODOまたはPRコメントとして残す。
